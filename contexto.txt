1. RESUMEN DEL PROYECTO
===============================================================================
HopeCare es un sistema de escritorio para gestión hospitalaria desarrollado en
Java con NetBeans. Permite gestionar pacientes, médicos, citas, consultas,
medicamentos, exámenes de laboratorio, facturación y un dashboard con métricas.
La base de datos es SQLite. Se aplican patrones de diseño sólidos para
mantener la separación de responsabilidades, la escalabilidad y la facilidad
de prueba.

===============================================================================
2. TECNOLOGÍAS Y ENTORNO
===============================================================================
- Lenguaje: Java 17+
- IDE recomendado: Apache NetBeans 18+ / IntelliJ IDEA
- Gestor de dependencias: Maven
- Base de datos: SQLite (driver xerial sqlite-jdbc)
- Interfaz gráfica: JavaFX 21 (FXML + CSS) y Swing (panel alternativo de citas)
- Logging: SLF4J con binding simple (opcional)
- Pruebas: JUnit 5 (en fase de implementación)
- Arquitectura: MVP + DAO + Fachada + EventBus (Observer)

Dependencias principales (pom.xml):
- sqlite-jdbc:3.45.1.0
- javafx-controls:21
- javafx-fxml:21
- slf4j-simple:2.0.9 (opcional)

===============================================================================
3. ARQUITECTURA GENERAL (CAPAS Y PATRONES)
===============================================================================
Se ha diseñado una arquitectura modular multicapa que combina varios patrones:

- Capa de presentación: JavaFX (FXML + controladores), Swing (CitasPanel) y
  consola (CitaConsoleView). Cada módulo tiene su propia carpeta 'view'.
- Capa de negocio: Presentadores (CitaPresenter, ConsultaPresenter), servicios
  y fachadas (GestionClinicaFacade).
- Capa de persistencia: DAOs (Data Access Objects) para SQLite con transacciones
  explícitas.

Patrones implementados:
- MVP (Modelo-Vista-Presentador) en citas_consultas con interfaces ICitaView e
  IConsultaView, permitiendo múltiples implementaciones de vista (JavaFX, Swing,
  consola).
- MVC (Modelo-Vista-Controlador) en módulos como registro de médicos.
- Patrón DAO para todas las entidades.
- Patrón Facade (GestionClinicaFacade) para coordinar farmacia y laboratorio.
- DTOs inmutables (FacturaDTO, DetalleFacturaDTO) para transferencia de datos.
- Patrón Observer (EventBus) para actualización reactiva del dashboard ante
  eventos como NuevaCitaEvent y NuevaFacturaEvent.
- Transaccionalidad explícita con commit/rollback en operaciones que abarcan
  múltiples tablas (registro de persona+medico, consulta+cita, receta+detalle).

5. DESCRIPCIÓN DE MÓDULOS
===============================================================================
- Registro: ABM de médicos y pacientes con herencia desde Persona. Transacción
  que guarda en persona y luego en medico/paciente.
- Citas: Agenda de citas usando horario_atencion (día_semana, hora_inicio,
  hora_fin, intervalo_minutos). El CitaPresenter genera bloques de tiempo
  disponibles, filtra los ocupados por citas existentes y reserva. Publica
  NuevaCitaEvent en EventBus al agendar. Múltiples vistas: JavaFX (CitasController
  con DatePicker + ComboBox), Swing (CitasPanel) y consola (CitaConsoleView).
- Consulta médica: El ConsultaPresenter carga citas PROGRAMADA, permite
  seleccionar una, registrar síntomas/diagnóstico/tratamiento (inserta en
  consulta y cambia estado cita a ATENDIDA en una transacción). Una vez
  guardada, permite solicitar exámenes de laboratorio y recetar medicamentos
  (ambos con manejo transaccional explícito). El ConsultaController (JavaFX)
  implementa la UI con diálogos modales para seleccionar exámenes y recetas.
- Farmacia: Entrega de medicamentos asociados a recetas, descuento automático
  del stock_actual. Usa fachada con control de roles.
- Laboratorio: Registro de resultados de exámenes asociados a solicitudes.
- Facturación: Agrupa consultas, exámenes y medicamentos pendientes de facturar
  (campo facturado = 0). Calcula subtotal, impuesto, total. Genera factura y
  detalles. Usa DTOs inmutables.
- Dashboard: Muestra citas del día, ingresos del mes, medicamentos con stock
  bajo. Se actualiza automáticamente con eventos (Observer).

6. BASE DE DATOS SQLITE
===============================================================================
Archivo: sisgeho.db (se crea automáticamente en la raíz del proyecto)

Principales tablas (script completo en sisgeho_schema.sql):
- persona (id_persona, documento_identidad)
- medico (id_medico, id_persona, id_especialidad, registro_medico)
- paciente (id_paciente, id_persona, historia_clinica)
- especialidad
- horario_atencion (id_medico, dia_semana, hora_inicio, hora_fin, intervalo_minutos)
- cita (id_paciente, id_medico, fecha_hora, estado)
- consulta (id_cita, diagnostico, sintomas, tratamiento, facturado)
- medicamento (id_medicamento, nombre, stock_actual, stock_minimo)
- receta, detalle_receta, entrega_medicamento
- examen_laboratorio, solicitud_examen
- factura, detalle_factura
- usuario, rol

7. CLASES Y COMPONENTES CLAVE
===============================================================================
- DatabaseConnection: Singleton para conexión a SQLite con registro explícito
  del driver.
- CrearBaseDatos: Ejecuta el script SQL para crear tablas.
- CargarDatosPrueba: Inserta datos mínimos (roles, especialidades,
  medicamentos, horarios, médico de prueba, paciente).
- Persona: Clase base con idPersona, documentoIdentidad, nombre y apellido.
- Medico extends Persona: Agrega idMedico, idEspecialidad, registroMedico y
  nombreEspecialidad (para display en tablas).
- Paciente extends Persona (nuevo): Agrega idPaciente, historiaClinica.
- MedicoDAO.insertarMedico(): Transacción que guarda en persona y luego en
  medico. También cuenta con listarTodos() que hace JOIN a persona y
  especialidad para obtener datos completos.
- PacienteDAO (nuevo): Método listarTodos() con JOIN a persona.
- CitaPresenter: Lógica de negocio para horarios: genera bloques de tiempo a
  partir de intervalo_minutos, filtra ocupados y publica NuevaCitaEvent.
- ConsultaPresenter: Coordina el flujo completo de consulta: carga citas
  pendientes, registra consulta (transacción), solicita exámenes y receta
  medicamentos con transaccionalidad explícita.
- CitaDAO: Métodos obtenerCitasPorMedicoYFecha (para filtrar horarios),
  insertarCita, actualizarEstado, obtenerCitasPorEstado,
  listarTodasConNombres (todas las citas ORDER BY fecha DESC con JOINs) y
  obtenerCitasPorEstadoConNombres (JOINs con persona para obtener nombres de
  paciente y médico).
- Cita: Modelo con campos pacienteNombre y medicoNombre para mostrar nombres
  en lugar de IDs en la UI de consultas.
- ConsultaDAO.insertarConsultaYActualizarEstado(): Transacción que inserta
  en consulta y cambia cita.estado a 'ATENDIDA' con commit/rollback.
- CitasController (JavaFX): Implementa ICitaView con tabla de citas existentes
  (ID, Paciente, Médico, Fecha/Hora, Estado) como vista principal. "Nueva Cita"
  abre un diálogo modal con MVP inline (ICitaView temporal + CitaPresenter). El
  diálogo contiene: buscador de pacientes con tabla filtrada, filtro de
  especialidad + buscador de médicos con tabla filtrada, selector de días
  disponibles (desde HorarioAtencionDAO.obtenerHorariosPorMedico()), auto-
  asignación de fecha a la próxima ocurrencia del día, selector de horarios y
  botón reservar. Al cerrar, refresca la tabla de citas.
- CitasPanel (Swing): Implementación alternativa de ICitaView en Swing (usa
  TextFields).
- ConsultaController (JavaFX): Implementa IConsultaView y delega toda la lógica
  de negocio a ConsultaPresenter (MVP completo). ComboBox de citas pendientes
  que muestra nombres reales de pacientes y médicos, TextArea para
  síntomas/diagnóstico/tratamiento, y diálogos modales (Dialog) para
  solicitarExamen (devuelve Integer) y recetarMedicamento (devuelve RecetaRequest).
- IConsultaView.RecetaRequest: Clase interna anidada para transportar datos
  de receta (idMedicamento, cantidad, dosis) desde la vista al presentador.
- FacturacionService.generarFactura(): Reúne conceptos pendientes, calcula
  totales, inserta factura y detalles, marca conceptos como facturados. Todo
  en una transacción.
- GestionClinicaFacade: Métodos procesarEntregaMedicamento y
  registrarResultadoExamen con validación de rol (FARMACIA, LABORATORIO).
- EventBus: Singleton que permite registro y publicación de eventos. Usado
  para refrescar el dashboard tras nueva cita o factura.
- DashboardDAO: Consultas agregadas a SQLite (COUNT, SUM) para métricas.

8. EVENTOS Y OBSERVER
===============================================================================
Eventos: NuevaCitaEvent, NuevaFacturaEvent
Publicación:
  - NuevaCitaEvent: desde CitaPresenter.reservarCita() tras insertar la cita.
  - NuevaFacturaEvent: desde FacturacionService.generarFactura().
Suscripción: DashboardController se suscribe en initialize() y actualiza los
             indicadores (citas del día, ingresos del mes, stock bajo) con
             Platform.runLater() para no bloquear la UI.

9. FLUJOS PRINCIPALES
===============================================================================
- Registrar médico → guardar en persona y medico (transacción).
- Agendar cita → CitaPresenter: seleccionar médico y fecha, consultar
  HorarioAtencionDAO, generar bloques según intervalo_minutos, filtrar
  ocupados vía CitaDAO.obtenerCitasPorMedicoYFecha(), mostrar disponibles
  en la vista, reservar (insertar cita + publicar NuevaCitaEvent).
- Atender consulta → ConsultaPresenter/Controller: cargar citas PROGRAMADA,
  seleccionar una, ingresar síntomas/diagnóstico/tratamiento, ejecutar
  ConsultaDAO.insertarConsultaYActualizarEstado() (transacción: INSERT
  consulta + UPDATE cita SET estado='ATENDIDA'), habilitar botones de
  solicitar examen y recetar medicamento.
- Solicitar examen → desde consulta guardada: diálogo modal lista
  exámenes disponibles (ExamenLaboratorioDAO), seleccionar, insertar
  SolicitudExamen (con Connection explícita).
- Recetar medicamento → desde consulta guardada: diálogo modal con
  ComboBox de medicamentos, Spinner de cantidad, TextField de dosis.
  Transacción explícita: INSERT receta + INSERT detalle_receta con
  commit/rollback.
- Farmacia → seleccionar receta y medicamento, entregar, disminuir stock
  (vía GestionClinicaFacade con validación de rol FARMACIA).
- Laboratorio → seleccionar solicitud pendiente, ingresar resultado.
- Facturación → FacturacionService.generarFactura(): reúne conceptos no
  facturados (facturado=0), calcula subtotal/impuesto/total, INSERT factura +
  detalles, UPDATE conceptos como facturados, publica NuevaFacturaEvent.
  Todo en una transacción.
- Dashboard → DashboardController suscrito a EventBus, recibe eventos y
  actualiza métricas con Platform.runLater().

INSTRUCCIONES PARA EJECUTAR EL PROYECTO
===============================================================================
1. Clonar repositorio o descargar código.
2. Abrir con NetBeans (o IntelliJ) como proyecto Maven.
3. Asegurar que el archivo pom.xml tenga las dependencias necesarias.
4. Ejecutar la clase principal: com.esperanza.hopecare.main.HopecareApp
   o usar comando: mvn javafx:run
5. La base de datos se inicializa automáticamente al primer inicio:
   - Si no existen tablas → ejecuta sisgeho_schema.sql
   - Si la BD está vacía → inserta datos de prueba (5 pacientes, 3 médicos,
     10 citas, medicamentos, exámenes, horarios)
   - Para resetear: borrar sisgeho.db de la raíz y reiniciar la app.
